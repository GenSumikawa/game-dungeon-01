# ランダムダンジョン（リアルタイム戦闘版）

シンプルなテキストベースのダンジョン探索ゲームです。  

---

## 概要

- ランダム生成されたダンジョンを探索する
- プレイヤーはゴール到達を目指す
- 敵は時間経過でバラバラに動く
- 敵には「向き」と「攻撃」があり、位置取りが重要

👉 回避だけでなく「戦う / 避ける / 位置を取る」という判断が必要なゲーム

---

## 実行方法

1. `index.html` をブラウザで開くだけ  （ローカルサーバ不要）
2. 「https://gensumikawa.github.io/game-dungeon-01/」にアクセス

---

## 操作方法

| キー | 動作 |
|---|---|
| Space | ゲーム開始 |
| W / A / S / D | 移動（向き変更） |
| 1 | 攻撃（前方1マス） |

---

## ゲームルール

- プレイヤー（○）をゴール（◎）まで移動すればクリア
- HPが0になるとゲームオーバー
- 壁（■）は通れない
- 敵と同じマスには入れない（重複しない）

---

## 戦闘仕様（重要）

### ■ プレイヤー

- 向きがあり、**前方1マスのみ攻撃**できる
- 攻撃はキー「1」

---

### ■ 敵

- **0.5秒ごとに行動（非同期）**
- 各敵が独立に
  - 20%で移動
  - 80%で待機

---

### ■ 敵の行動優先順位

1. 正面にプレイヤーがいる  
   → **攻撃を優先（移動しない）**

2. プレイヤーが正面にいない  
   → ランダム移動（確率20%）

---

### ■ 敵の攻撃

- 正面1マスにプレイヤーがいる場合のみ
- 確率でダメージ発生
- 1回攻撃すると**クールタイムあり（連続攻撃不可）**

👉 理不尽な連続ダメージを防止

| ◎ | ゴール |
| △ ＞ ▽ ＜ | 敵（向き付き） |

| 　| 床（空白） |

## 技術構成

- HTML + CSS + JavaScript（バニラ）
- 描画：`<pre>` タグによるテキスト表示
- アルゴリズム：
  - BFSによる到達可能チェック
  - ランダムマップ生成
  - タイマーによる非同期敵AI

---

## 設計ポイント

### 1. 非同期ゲームループ
- `setInterval` による時間ベース処理
- プレイヤー入力と敵行動が独立

---

### 2. 位置取りゲーム化
- 向き + 前方攻撃
- 重複禁止（敵と同じマスに乗れない）

👉 「避ける」だけでなく  
👉 「読む」ゲームになる

---

### 3. 理不尽さの制御
- 敵攻撃にクールタイム
- 1tickで1回のみ被弾
